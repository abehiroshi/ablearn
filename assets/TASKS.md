# 外部制作タスク・素材タスク

アプリ実装とは別系でやり取りするタスクの一覧。2方向ある:

- **T系（外部制作）**: アプリ側が依頼 → 外部が制作して納品（画像・選定リンクなど）
- **S系（素材→問題化）**: オーナーが学校のテスト・小テスト・ワーク等の写真を Google ドライブに置いて依頼 → 開発側が問題化

## 運用ルール（共通）

- 1タスク = 1セクション。新しいタスクは末尾にテンプレートをコピーして追加する
- ID は `T-001` / `S-001` から連番。欠番を再利用しない
- 状態は3段階: **依頼中** → **納品済み**（取り込み待ち）→ **取込済み**（アプリに反映済み）
- 判断に迷ったらメモ欄に質問を書いて止める
- 将来の分まで先回りで依頼を積まない。必要になったタイミングで同じルールで追加する

## S系の運用ルール

- 写真は Google ドライブの共有フォルダ **「Ablearn入口」** に置く
  （URL: https://drive.google.com/drive/folders/1poY3icOjTvfcOKa7LylIA15aWt4stTyu ／ フォルダID: `1poY3icOjTvfcOKa7LylIA15aWt4stTyu`。開発側はDrive連携でこのIDから読む）
- ファイル名か添付メモで「教科・いつの・何か」（例: 数学・6/10小テスト）がわかるようにする
- **複数枚で1つのまとまり（1セット）にしたい場合はサブフォルダに入れる**（サブフォルダ名がそのままセットの呼び名になる。例: `数学0610小テスト/` に3枚）
- **写真はリポジトリにコミットしない**（このリポジトリは public＝公開のため）。元ネタはドライブに留める
- 収録する問題は**アレンジ版（同じ概念を確認するオリジナル問題）**として作成する。
  定期テスト・市販ワークの問題をそのまま転載しない（公開リポジトリでの著作権配慮）
- 開発側はドライブから画像を読み、概念・出題形式・難易度を抽出して変種を生成 →
  `npm run validate:content` を通してコミット → 状態を取込済みに更新
- 生成セットには出典メタデータ（計画21の `origin`）を付け、Library で由来がわかるようにする

## S系テンプレート

```
## S-0XX: タイトル（例: 数学 6/10小テストの問題化）

- 状態: 依頼中
- 元ネタ: ドライブのファイル名（複数可）
- 教科・単元: 
- 希望: アレンジの強さ・問題数・形式など（任意。無指定なら開発側判断）
- メモ:
```

## T系テンプレート

```
## T-0XX: タイトル

- 状態: 依頼中
- 内容: 何を作るか1〜2行
- 仕様: サイズ・形式・構成など、制作に必要な条件
- 参照: 参考にする既存ファイルや資料
- 納品先: コミットするパス
- 取込手順: 納品後にアプリ側が行うこと
- メモ:
```
