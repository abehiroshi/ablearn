# コンテンツ生成手順書（概念ファミリー）

AIセッションでコンテンツを生成するときのプロンプトテンプレートと手順。
このファイルだけを読めば、セッションの文脈なしで同じ品質で生成できることを目的とする。
スキーマの正は [src/types.ts](../src/types.ts)、規約の正は [README](../README.md)（コンテンツの追加・更新）と
[docs/spec.md](spec.md)。

## 入力（依頼者が渡すもの）

```
教科: （例: 理科）
単元: （例: 化学変化と原子・分子）
範囲: （教科書の該当箇所・扱う概念のメモ。例: 化学反応式の係数合わせまで）
links候補: （親が選定した動画URL等。無ければ「なし」）
```

## 生成プロンプトテンプレート

以下をそのまま依頼文にする（[ ] を埋める）:

> Ablearn のコンテンツを生成して。docs/content-generation.md の手順に従うこと。
> 教科=[教科] 単元=[単元] 範囲=[範囲]。
> 範囲内の概念を列挙し、概念ごとに「概念ファミリー」を作って
> `public/content/chugaku/[教科dir]/` のセットJSONにまとめ、index.json に登録、
> `npm run validate:content` を通して結果を見せて。

## 概念ファミリーの構成

**1概念 = 確認したい知識1つ**（例:「同類項をまとめられる」「化学反応式の係数を合わせられる」）。
概念ごとに次を作る:

| 要素 | 内容 |
| --- | --- |
| 変種（stage0） | choice（4択）。`answers[]`（受理表記）を併記して input 形式でも出せるようにする |
| 変種（stage1） | input（自力入力）。同じ概念の別の数値・別の題材 |
| 変種（stage2） | `difficulty: 3` の応用（choice か input） |
| 共通 | 同じ `concept` ID を付与（命名: `教科略-単元略-概念`。例: `sci-kagaku-keisuu`） |

- 各問題に `explanation`（なぜそうなるか1〜2文）と `hints`（弱→強の2段階。
  1段目=考えるポイント、2段目=途中までの解き方）と `difficulty`（1/2/3）を付ける
- アプリ側の挙動: 同じ `concept` の変種は習熟度の段に合った**1問だけ**が出題され、
  `answers` つき choice は習熟が進むと自動で input 形式に変わる（だから受理表記の網羅が大事）
- ID規約: `concept` は「確認したい知識」単位（1単元に複数あってよい）。セットIDは
  `教科略-ファイル名` に揃える。教科ディレクトリ名は `index.json` の既存構成
  （math / english / science / social / japanese / music / art / pe / tech-home）に従う
- 単元に1本、`kind: "lesson"` のレッスン（解説カード＋易しい問題のステップ列）を作る。
  順序は「説明 → すぐ確認」の交互。正解は本文で先に教えてから問う
- 変種は**作成時に**作る（実行時生成はしない）

## 品質規約（READMEの注意に加えて）

- choice の誤答選択肢は「よくある間違い」から作る（符号ミス・係数の数え間違い等）。
  でたらめな選択肢は学習効果がない
- input の `answers` は漢字・ひらがな・別表記・順序違い（`2x+6y` と `6y+2x`）を網羅する
- order は「別の正しい並べ方」が存在しない文だけ（チャンク化 `"to play"` で一意にできる）
- **links は親が選定したものだけ**。生成時は「候補の提案」までに留め、
  index.json への登録は選定済みURLのみ
- 著作権: 教科書・市販ワーク・学校テストの問題をそのまま転載しない。
  同じ概念を確認する**オリジナルのアレンジ問題**として作る（公開リポジトリのため）

## 検証 → レビュー → 登録

1. `npm run validate:content` … スキーマ・id一意・answer範囲・形式チェック（必須）
2. `node scripts/validate-content.mjs --links` … リンクの死活（links を足したときだけ）
3. **Breaker レビュー**で内容の正確性（計算・事実・誤答選択肢の妥当性）を確認する。
   検証は形式までしか見ない（[計画14](plans/14-content-pipeline.md) の非スコープ）
4. `public/content/chugaku/<教科>/` に配置し、`index.json` の該当単元に登録して push
   （CIゲートが再検証する）

## 実例

最初の実例: `public/content/chugaku/science/kagaku-hannoushiki.json`
（化学反応式の係数・1概念ファミリー＋レッスン）。形の参考にする。
