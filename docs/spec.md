# 不変条件（spec）

コードから読み取りにくい約束だけを書く。画面の見た目・操作手順は書かない。
ここに書かれた不変条件は `src/lib/*.test.ts` の自動テストが守る。

## XP

| 場面 | XP |
| --- | --- |
| 1発正解（choice/input/order） | +10 |
| セッション内リトライで正解 | +5 |
| ヒントを使った正解 | +5（リトライ正解と同額。新区分を増やさない） |
| フラッシュカード「覚えた」 | +5 |
| レッスン内の正解 | +2（採点プレッシャーを下げる） |
| 写経の完了 | +2（見ながら打てただけ。覚えた証明ではない） |
| 模擬テストの正解 | 通常と同じ（1発正解扱い） |
| 「わからない」 | 0（減点もなし。取組数にはカウントする） |

## localStorage 後方互換の約束

- キーは `ablearn:<コレクション>:v1`（例: `ablearn:chugaku:v1`）。コレクション間で進捗は完全に独立し、
  横断の共有要素（合算streak等）は持たない
- 読み込みは常に `{ ...emptyState(), ...保存データ }` のマージで行い、
  **古い形式のデータを読んでも壊れない**
- スキーマ（AppState）にフィールドを追加するときは、変更前の形式を
  `src/lib/__fixtures__/appstate.ts` にフィクスチャとして追加する
- コンテンツの問題 `id`・セット `id` は変更しない（進捗が `セットID/問題ID` で保存されるため、
  変えると履歴が切れる）
- 進捗のキーは出題形式に依存しない（choice を input 形式で出しても同じ問題として記録される）

## モードの導出

- **テストモード** = テストが登録されていて、かつ最終日（時間割の最大日付）を過ぎていない。
  最終日当日まではテストモード。手動切替は存在しない
- 最終日を過ぎたら労い表示＋登録クリアの促し。クリアまたは未登録なら**日常モード**
- テストモード中、その日の試験が終わった教科はおすすめから外す

## おすすめの並び

- 共通の並び: 未挑戦 → ベストスコアが低い順
- テストモードは「試験日が近い教科」の重みを先に掛ける
- 日常モードの対象は進行中単元。何もなければ全体から1件（未挑戦 → 最後に解いたのが古い順）

## 模擬テスト

- 出題は範囲（テスト登録時はその範囲、未登録時は選択した単元）からランダム。
  レッスンセットは出題しない
- 難易度バランス: 応用（difficulty 3）約3割。足りなければ反対側から補充
- 実施中は正誤・解説・ヒントを一切見せない。制限時間切れで自動採点
- 中断時は結果を保存しない（解答済みの記録は通常どおり残る）

## レッスン

- 線形進行・リトライなし。間違えてもその場で解説して次へ
- レッスンの解答は `questionStats`（復習リスト）に入れない。解答履歴（history）には入る
- 完走で `setRecords` に記録（Library では「完了/未学習」表示）

## 解答履歴（history）

- 「日付 → 問題キー → {正解数, 不正解数, わからない数, ヒント使用数, 解答時間合計ms}」の日次集計のみ。
  生ログ（1解答1レコード）は持たない
- リトライを含むすべての解答を加算する。解答時間は問題表示から解答確定まで
- 「わからない」は不正解と**区別して**記録する（習熟度推定・つまずき検知が利用する）。
  復習リスト（questionStats）上は不正解として扱う。模擬テストには「わからない」を出さない

## 習熟度（概念の段位）

- 段は「みながらかける(-1・写経) → 4択でできる(0) → 自力でとける(1) → 応用もできる(2)」。
  concept 単位で昇降格する。保存データの level 0〜2 の意味は計画25の前後で変わらない（後方互換）
- **昇格**: ヒントなし正解が2回連続、かつ前回のヒントなし正解から中2日以上
  （`PROMOTE_GAP_DAYS = 3`。定数は `src/lib/mastery.ts`）
- **降格**: 不正解 / わからない / ヒントを最後まで使った正解。ヒント途中の正解は段維持・連続リセット。
  choice 段（0）からは連続2回（`DEMOTE_TO_TRACE_WRONGS`）で写経段へ落ちる（1回では落ちない）
- **写経段（計画25）**: 解答実績が全く無い初見の概念も写経段から始まる。
  answers を持つ最も易しい変種（difficulty 3 を除く）を「正解のゴーストを見ながら打つ」形式で出題し、
  完了すると choice 段へ進む（正解の連続には数えない・翌日に再確認）。
  写経にできる変種が無い概念（flashcard / order のみ等）は choice 段の出題へフォールバック
- 写経の完了は `questionStats`（復習リスト・達成度）に入れない。解答履歴（history）には入る
- 解答時間は評価に使わない（速さを評価すると雑に答える誘因になる）
- **昇降格の証拠は提示形式ではなく解答形式ベース**（計画34）: 現在の段が要求する形式より
  下の形式に切り替えて解いた正解は、その段の昇格の連続に数えない（降格もしない）。
  例: input 段の問題を choice に切り替えて正解 → choice 段の証拠どまり・習熟度は実態どおりに止まる
- 正解で定着確認日（dueDate）を段に応じて先送り（3/7/14日。写経段は choice 段と同じ）。
  期日が来たセットは日常モードのおすすめで最優先（「定着チェックの時期だよ」）
- 習熟度を更新するのは通常クイズと復習の解答（模擬テスト・レッスンは更新しない）
- concept 未設定の問題は従来どおりの出題・記録（後方互換）

## 数字の体系と節目（祝福）

- メトリクスはすべて**単調増加**（絶対に下がらない）。達成度 = 一度でも正解した問題数 ÷ 全問題数
  （分母にレッスンは含めない。「いま解けるか」は習熟度（段位）の役割）
- 節目: 解答数 100/300/500/1000…、streak 3/7/14/30…、学習日数 10/30/50…、達成度 25/50/75/100%
- **祝福は跨いだ瞬間に一度だけ**。祝福済みIDを `celebrated` に記録して再表示しない
- 解答の流れを止めない: 解答中は軽いチップ、結果画面でまとめて表示。
  模擬テスト中は表示しない（正誤が漏れるため。結果画面でまとめる）
- テスト範囲の達成度は表示のみ（テストごとに範囲が変わるため節目は作らない）
- スキンは節目との直接対応で解放（ショップ・XP消費にしない）。解放状態は celebrated から導出し、
  選択は `selectedSkin`。シート未納品のスキンはメインのポーズで代用される

## バックアップ

- エクスポートは `{ app: "ablearn", schemaVersion, collection, exportedAt, state }`。AppState は不透明に全量入出力
- 別コレクションのバックアップはインポートを拒否する（collection 無しの旧バックアップは通す）
- インポートは外形検証（app名・バージョン・xp/dailyLogの型）のみ。サマリ確認なしに上書きしない
- 未知の（大きい）schemaVersion は拒否、古い形式は emptyState マージで補完

## コンテンツ

- スキーマ準拠・id 一意は `npm run validate:content` が守る（CIゲートに含まれる）
- 教科の color / icon は重複させない
- links の URL は http(s) のみ。親が選定したものだけを入れる
