# 模擬テスト（定期テスト風の挑戦）

- 状態: 未着手
- 触るファイル範囲: `src/` のみ
- 引き継ぎメモ: なし

## 目的

テスト直前の仕上げとして、本番に近い緊張感で実力を測る。

## 内容

- 範囲（テスト登録済みならその範囲、なければ単元選択）からランダムN問を出題
- 制限時間つき。デフォルト20分、開始時に変更可（本番より短い「仕上げ用」の位置づけ）
- 途中で解説・正誤を見せない。終了後に100点換算の結果＋全問の見直し
- 結果は記録として保存（挑戦日時・点数・範囲）。ベストスコア表示
- テストモード（[01](01-modes-and-test-schedule.md)）中はホームに「模擬テストに挑戦」の導線を出す

## 非スコープと破綻条件

- 難易度バランスの出題構成（基本7割＋応用3割など）は [07](07-difficulty-and-hints.md) のタグ付け完了後に有効化する。それまでは単純ランダム
- 過去の模擬テストの問題単位の振り返りはしない（解答履歴 [04](04-answer-history.md) には通常どおり記録される）

## 受け入れ条件

- 制限時間が切れると自動で採点に進む
- 模擬テスト中はヒント・解説が一切見えない
- 中断（画面を閉じる）してもアプリの他の進捗が壊れない
