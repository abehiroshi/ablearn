# タブレットレイアウト（全画面の大画面対応）

- 状態: 未着手
- 触るファイル範囲: `src/styles.css`・各 `src/screens/`（構造変更は最小限）
- 引き継ぎメモ: クイズ画面のタブレット対応は [05](05-scratch-pad.md) で先行済みの想定

## 目的

主端末がタブレットのため、スマホ前提の1カラムを大画面で使いやすくする。
スマホ利用も継続するためレスポンシブで両立する。

## 内容

- ホーム: 2カラム化（おすすめ・カウントダウン系と教科一覧の並置）
- Library / Stats: 幅を活かしたグリッド表示
- タッチターゲットの拡大、余白・文字サイズの段階設計（ブレークポイントを1つ決めて全画面で統一）

## 非スコープと破綻条件

- 横画面スマホへの最適化はしない → 横スマホで常用したくなったら調整
- PC・マウス向けの最適化はしない（崩れなければよい）

## 受け入れ条件

- タブレット縦・横、スマホ縦の3パターンで全画面が崩れない
- スマホの操作感が現状から劣化しない
