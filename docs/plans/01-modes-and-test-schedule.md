# 日常モードとテストモード（時間割つき逆算）

- 状態: 未着手
- 触るファイル範囲: `src/` のみ（`public/content/` には触れない）
- 引き継ぎメモ: なし

## 目的

アプリを「問題集を自分で選んで解く」から「開けば今日やることが出ているコーチ」に変える。
テストが近くない期間も毎日使えるようにする。

## 機能

### 日常モード（基本状態）

- 教科ごとに「いま授業でやっている単元」を選択できる（進行中単元）
- ホームに「今日のおすすめ」を1〜3個提示:
  進行中単元のセットを「未挑戦 → ベストスコアが低い順」、＋全範囲の復習対象（直近不正解）の件数

### テスト登録

- 「次のテスト」を1件だけ登録できる: テスト名・日別時間割（日付＋その日の教科の並び）・教科ごとの範囲（セット選択）
- 範囲選択は、進行中単元としてマークされてきた単元が最初からチェック済みの状態で始まる（下書き）
- 登録は子ども本人が行える前提のUI（日を追加→教科をタップで選ぶ）

### テストモード

- 「テストが登録されていて最終日を過ぎていない」なら自動的にテストモード表示。手動切替は作らない
- ホームにカウントダウン＋日別時間割を表示。教科ごとに残り日数が見える
- おすすめは日常モードと同じエンジンで、対象をテスト範囲に、重みを「試験日が近い教科を優先」に変える
- テスト期間中、終わった日の教科はおすすめから外す
- 最終日を過ぎたら労い表示＋登録クリアの促し→日常モードへ自動で戻る

## データ

`AppState` に2フィールド追加（`emptyState` マージで後方互換を維持）:

- 進行中単元のID群（教科ごと）
- テスト1件（名前・日別時間割・教科別範囲セットID群）

## 非スコープと破綻条件

- 複数テストの同時管理はしない → 別範囲のテストを同時期に追いたくなった時点で破綻する。定期テスト用途なら当面1件で足りる
- 時限・開始時刻は持たない → 優先度に効くのは「どの教科がどの日か」だけ。使い道が出たら足す
- 残り日数からの学習量自動配分・SRS（忘却曲線）はやらない → おすすめエンジンの内側に後から差し込める

## 受け入れ条件

- テスト未登録時: 進行中単元を選ぶと、ホームのおすすめがその単元から出る
- テスト登録時: カウントダウンと時間割が表示され、試験日が近い教科が優先して提案される
- テスト期間中: 終了した日の教科は提案されない
- 最終日経過後: 日常モードに自動で戻り、進行中単元ベースの提案が再開する
- 既存ユーザーの localStorage データ（XP・streak・成績）が壊れない
