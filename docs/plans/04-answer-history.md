# 解答履歴の記録（成長グラフの基盤）

- 状態: 完了（2026-06-12）
- 触るファイル範囲: `src/lib/storage.ts`・各画面の解答確定処理
- 引き継ぎメモ: `AppState.history` に日次集計を記録開始。`hints` は予約フィールドで常に0（[07](07-difficulty-and-hints.md) 実装時に `recordHistory` の `hints` 引数へ渡す）。Breaker 指摘の major 2件（StrictMode での ref リセット・時計巻き戻し）は検証の結果いずれも誤検出と判断

## 目的

成長グラフ・習熟度分析（[09](09-visualization.md)）の元データ。過去には遡れないため、
表示機能より先に記録だけを開始する。実装キューの最優先。

## 内容

- `AppState` に履歴を追加: 「日付 → 問題キー → {正解数, 不正解数, ヒント使用数, 解答時間合計ms}」の日次集計
- 生ログ（1解答1レコード）は持たない。日次集計で成長グラフに必要な粒度は足りる
- 解答時間は問題表示から解答確定までを計測。flashcard も対象（めくって自己申告まで）
- ヒント使用数のフィールドは予約しておく（記録開始は [07](07-difficulty-and-hints.md) 実装後）

## 非スコープと破綻条件

- 表示はしない（09で作る）
- 日内の解答順序は持たない → 「何問目で集中が切れるか」のような時系列分析をしたくなった時点で破綻する

## 容量の目安

1日50問解いても1日数KB。localStorage の制限（約5MB)に対して数年分でも問題ない規模。

## 受け入れ条件

- 解答するたびに当日の履歴が更新される
- 既存の localStorage データが壊れない（`emptyState` マージ）
- バックアップ（[02](02-progress-backup.md)）のエクスポートに含まれる
