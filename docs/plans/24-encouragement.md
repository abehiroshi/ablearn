# 誤答時・停滞時の励まし演出

- 状態: 未着手
- 触るファイル範囲: `src/screens/QuizScreen.tsx`・`src/lib/`（文言定義）
- 引き継ぎメモ: 根拠は [調査レポート](../research/learning-apps-survey.md) B-4（Brilliant の励まし演出）

## 目的

祝福（うまくいった時・18）の逆側を埋める。つまずいている時に責めずに励まし、
レッスン誘導（13）へ自然につなぐ。既存の Abler ポーズ（ganbare / uun / kangaechu）で素材ゼロ。

## 内容

- つまずき検知（13の閾値）に達したとき、誘導の前段として Abler の励まし（ポーズ＋短い一言）を出す
- 誤答そのものには毎回演出しない（うるさくなる）。発動は 13 と同じ「セッション内・同一キー1回」の抑制を流用
- 文言は複数パターンからランダム。「責めない・次の一歩を示す」トーンで統一
  （例:「むずかしいよね。ヒント見てみる？」「ここつまずきやすいんだ。レッスンで整理しよう」）
- 模擬テスト中は出さない（本番の緊張感を保つ）

## 非スコープと破綻条件

- 専用アニメーション・効果音は作らない（素材が来たら差し替え）
- 励ましの効果測定はしない → 鬱陶しがられている兆候（誘導の無視率など）が見えたら頻度を下げる

## 受け入れ条件

- つまずき検知時に励まし→誘導の順で表示される
- 同じ概念で2度目以降のセッション内表示はない
- 模擬テスト中に出ない
