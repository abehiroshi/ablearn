# 前提概念への遡り（v1: コレクション内限定）

- 状態: 未着手
- 触るファイル範囲: `src/types.ts`・`src/lib/`（recommend / content）・`scripts/validate-content.mjs`・`docs/content-generation.md`（前提宣言の項）・`public/content/chugaku/`（数学の前提タグ付け）
- 引き継ぎメモ: 根拠は [調査レポート](../research/learning-apps-survey.md) B-2（Qubena の遡行学習）。v1 はコレクション内限定（オーナー決定 2026-06-12）

## 目的

つまずきの原因が手前の概念にある場合（例: 一次関数のつまずき ← 比例・方程式）、
その概念の復習・レッスンへ誘導する。つまずき誘導（13）のカリキュラム横断への拡張。

## 内容

- **前提の宣言**: コンテンツに概念のメタ定義を持つ場所を作る
  （`public/content/<col>/concepts.json`: id・表示名・prerequisites[]）
- **誘導の拡張**: つまずき検知（13）の誘導先に「前提概念の復習」を追加。
  同一単元のレッスンより前提概念の習熟が低い場合は前提を優先して提示する
- **v1 の制約**: 遡り先はコレクション内に存在する概念のみ。存在しない前提（中1範囲など）は
  宣言してよいが誘導には使わない（リンク切れを表示しない）
- 検証スクリプトに「未定義 concept への前提参照」「循環参照」の検出を追加
- 生成手順書に前提概念の宣言の項を追加
- 数学の既存概念に前提タグを先行付与（連立方程式←式の計算、など）

## 非スコープと破綻条件

- 中1範囲の振り返りコンテンツは作らない → 「前提が中1にある」つまずきが実運用で目立ったら、
  振り返り用ミニコンテンツ（または中1コレクション）を別計画で起こす
- 前提グラフの自動推定はしない（宣言ベースのみ）

## 受け入れ条件

- 前提概念の習熟が低い状態でつまずくと、前提の復習への誘導が出る
- 前提がコレクション内に無い場合は従来どおりの誘導（リンク切れが見えない）
- validator が未定義参照・循環参照を検出する
- 数学の主要概念に前提が宣言されている
