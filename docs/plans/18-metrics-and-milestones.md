# 数字の体系・達成度・チェックポイント祝福

- 状態: 完了（2026-06-12）
- 触るファイル範囲: `src/lib/`（metrics / milestones 新規）・`src/screens/`（Stats・Home・Library）・`src/lib/storage.ts`（祝福済み節目の記録）
- 引き継ぎメモ: 実装は `src/lib/milestones.ts`（節目検出は純粋関数・10テスト）。達成度の分母は起動後に全セットをバックグラウンド読込（失敗時1回リトライ。読込完了前の初正解で跨いだ達成度節目は祝福されない小さな競合が残る＝許容）。テスト範囲の達成度は表示のみで節目なし（範囲がテストごとに変わるため）。模擬テスト中は結果画面でまとめて祝福。「ヒントなし正解の累計」は日次集計から正確に出せないため未実装（マスター概念数で代替）。バッジの専用デザインは素材が来たら差し替え（今は絵文字チップ）

## 目的

「増えることがうれしい数字」を体系的に用意し、節目で褒めるタイミングを作る。
やれば上がる・習熟すれば上がる・幅を広げれば上がる・100%に向かう、の4系統。

## 数字の体系（すべて単調増加 = 絶対に下がらない）

| 系統 | 数字 | データ源 |
|---|---|---|
| 努力系 | 累計解答数・週の取組回数（全体/教科/単元）・累計学習日数 | 04の履歴 |
| 習熟系 | マスター済み概念数・ヒントなし正解の累計 | 12の段位（12完了後に有効化） |
| 幅系 | 取り組んだ単元数・教科数 | 既存実績 |
| 達成度系 | 単元/教科/テスト範囲の達成度% | 下記定義 |

- **達成度の定義**: 一度でも正解した問題数 ÷ 対象の全問題数。
  「いま解けるか」は習熟度（段位）の役割で、達成度は下がらない（達成感を壊さない）
- 表示は階層化: ホームは文脈に合う2〜3個だけ、Stats に全部＋バッジ棚、Library の単元一覧に達成度バー

## 節目（チェックポイント・キリ番）と祝福

- 達成度型: 25 / 50 / 75 / 100%。単元→教科→範囲の入れ子で、小さい祝福が頻繁に・大きい祝福が稀に来る
- 累積型: 解答数 100/500/1000…、streak 3/7/14/30日、学習日数 10/50/100日（後ろほど間隔を広げる）
- **祝福は跨いだ瞬間に一度だけ**。祝福済みの節目を進捗データに記録して再表示しない
- **解答の流れを止めない**: 解答中は軽いトースト＋Ablerポーズ、大きな祝福は結果画面でまとめて演出
- 節目検出は「解答前後の状態を比べて新たに跨いだ節目を列挙する」純粋関数にする（15のテストで検証）
- バッジ・称号: 達成の種類ごとにデザインし、Stats の棚に収集する

## 非スコープと破綻条件

- スキン解放は [19](19-skin-rewards.md)。XPで買うショップ制はやらない（XPは純粋なスコアのまま）
- 祝福演出は CSS アニメーション＋既存 Abler ポーズの範囲。専用イラスト・音は素材が用意されたら

## 受け入れ条件

- 同じ節目の祝福が二度出ない
- 達成度がどの操作でも下がらない（不正解しても%は維持）
- 節目検出のテストがある（境界値: ちょうど100問目・25%ちょうど等）
- 祝福済み記録がバックアップ（02）に含まれる
