# 漢検 準2級コンテンツ増強（量と質）

- 状態: 未着手
- 触るファイル範囲: `public/content/kanken/`（セット追加・index.json）・`scripts/`（カバレッジ集計の新規スクリプト）・`package.json`（scripts エントリ）・`docs/content-generation.md`（必要なら追記のみ）。カタカナ正規化をやる場合のみ `src/lib/quiz.ts`＋テスト
- 引き継ぎメモ: なし

## 目的

計画38で作った「全分野一巡の骨格」を、準2級対策として実用になる量と質に引き上げる。
配点の大きい分野（書き取り50点・読み30点・四字熟語30点）から優先的に増強する（オーナー指示 2026-06-12）。

## 内容（優先順）

1. **書き取り・読みの増強**: 「基本2」「基本3」を追加し、書き取り計40問以上・読み計36問以上にする。
   語の選定は準2級配当漢字（高校在学程度の新規約328字）を優先し、頻出とされる定番語を中心にする
2. **四字熟語の質の補強**: 準2級配当字を含む四字熟語中心の「四字熟語（書き）基本2」を追加（10問以上）。
   既存の基本1（3〜4級レベル寄り）は導入用としてそのまま残す
3. **配当漢字カバレッジの可視化**: `scripts/kanken-coverage.mjs` を新設。準2級配当字リストを
   スクリプト内に持ち、`public/content/kanken/` の全問題文・解答から使用済みの配当字を抽出して
   「出題済み n/328・未出題の字一覧」を出力する。以後のセット追加はこの出力を見て埋める。
   - 配当字リストは生成時にBreakerレビュー対象とする（リスト自体の誤りが最大のリスク。
     出典メモをスクリプト冒頭に残し、「参考値」であることを明記する）
   - CIゲートには入れない（カバー率は品質ゲートではなく作業の道具）
4. **残り6分野の「基本2」**: カバレッジ出力を参照しながら各分野8〜10問を1セットずつ追加
   （部首・熟語の構成・四字熟語の意味・対義語類義語・同音同訓・誤字訂正・送りがな）
5. **（小・任意）カタカナ入力の救済**: 読み問題でカタカナ入力が不正解になる件。
   `normalizeAnswer`（src/lib/quiz.ts）にカタカナ→ひらがな正規化を入れる場合は、
   chugaku 側の既存問題（カタカナ表記が正解の理科用語等）で「ひらがな解答も正解扱いになる」
   挙動変化を許容できるか先に確認し、互換テストを追加してから入れる。
   判断に迷ったら見送ってよい（問題文の「ひらがなで」明示で運用できている）

## 生成・検証の規約

- 生成は `docs/content-generation.md` の「漢検コレクション（kanken）の注意」に従う
  （形式の使い分け・誤字訂正の同音一字・四字熟語の実在確認・転載禁止）
- セット追加ごとに `npm run validate:content` ＋ 漢字知識のBreakerレビュー
  （計画38の実績: 部首の覚え方・誤字訂正の同音前提・配当レベルで指摘が出た。同じ観点で見る）

## 非スコープと破綻条件

- 本番形式の模擬テスト（60分・200点）はやらない → 受検日が決まったら
  模擬テスト（06）のコレクション対応として別計画を起こす
- 配当漢字328字の完全カバーはこの計画の完了条件にしない（カバレッジは道具であって目標ではない。
  完全網羅を目標にすると低頻出語の低品質問題を量産しがち）
- 他の級（4級・3級・2級）のコンテンツは対象外（計画38の方針のまま）
- 頻出度データそのもの（市販問題集の頻出表など）はリポジトリに取り込まない（著作権。
  語の選定の参考にするだけ）

## 受け入れ条件

- 書き取り3セット・読み3セット・四字熟語（書き）2セット以上が登録され、validate:content が通る
- 残り6分野に「基本2」が1セットずつある
- `node scripts/kanken-coverage.mjs` が「出題済み n/328＋未出題字一覧」を出力する
- 追加分すべてに漢字知識のBreakerレビューを実施し、major 0 で完了
- chugaku の進捗・既存テストに影響がない（カタカナ正規化を入れた場合は互換テストを追加）
