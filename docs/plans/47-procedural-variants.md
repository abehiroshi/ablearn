# 計算系概念の変種深掘り（数字替えの量稽古を10変種以上に）

- 状態: 未着手
- 触るファイル範囲: `public/content/chugaku/{math,science}/`・`public/content/chugaku/concepts.json`・
  `scripts/validate-content.mjs`（procedural 閾値の追加のみ）。アプリコード（pickVariant 等）には触れない
- 引き継ぎメモ: 2026-06-13 の監査結果に基づく。反復の両面のうち**機構は揃っている**
  （同じ問題=再戦・計画30／違う問題=変種ローテーション・計画40＋SRS・計画12）。
  不足はプールの深さのみ: 演習つき概念の大半が品質ゲート下限の**変種5丁度**（最大でも10）。
  SRS の復習周回＋再戦が回ると 5 変種では数周で「答えを覚えた」状態に戻る。
  計算ミスを減らす数字替えの量稽古は、知識系より深いプールが必要。

## 目的

計算手続き型の概念（解き方は同じで数字だけ違う）は、変種演習の量をこなすことが
計算ミス削減に直結する。知識系概念（5変種で十分）と要求量が違うため、
概念に「計算系」の区分を導入し、計算系だけ変種下限を 10 に引き上げて作り込む。

## 内容

- **concepts.json に `kind: "procedural"` を追加**: 対象は
  - 数学: 全計算概念（同類項・単項式・等式の変形・連立基本/文章題・一次関数の変化の割合/式/変域/交点・
    内角外角の計算・確率の計算・四分位範囲 ※46で新設の計算系も含む）。
    証明の穴埋め・合同条件の選択など判断系は対象外
  - 理科: 湿度計算・オームの法則・単位換算・質量保存/質量比・化学反応式の係数・電力/電力量（46で新設）
- **対象概念の変種を10以上へ増強**: d1〜d2 中心の数字替え。choices には実際の計算ミス
  （符号ミス・移項ミス・係数の掛け忘れ等）を誤答として埋め込む。explanation・2段ヒントは
  変種ごとに数値を反映した記述にする（使い回しの整合崩れを防ぐ）
- **validate-content.mjs に procedural 閾値を追加**: `kind: "procedural"` の概念は
  変種10以上・input可能4以上をエラーとして検査（知識系は現行の5/2のまま）

## 非スコープと破綻条件

- **ランタイムでの数字ランダム生成はしない**。静的 JSON の設計を守る。動的生成は誤答選択肢・
  解説・ヒントの整合を機械的に保てず、品質ゲートも通せない
- pickVariant・SRS のロジックは触らない（プールが深くなるだけで動作は同じ）
- 破綻条件1: 変種を機械的に量産すると explanation / choices と数値の不整合が混入しやすい。
  validate の整合チェック（answer と choices の対応・answers 表記）を通すことを各セットで確認する
- 破綻条件2: 既存問題の id 変更は履歴を切る。追補のみ（計画41の規約）
- 破綻条件3: 1セットの問題数が増えすぎると1回の演習が重くなる。セット分割が必要なら
  「基本/もっと練習」の2セット構成にし、index.json の同一単元内に並べる

## 受け入れ条件

- procedural 指定の全概念が変種10以上・input可能4以上をエラーモードで通過する
- 知識系概念の閾値（5/2）は変わらない（既存セットの再改修を発生させない）
- 既存問題の id 変更ゼロ
- 既存テスト＋追加分が全通過
