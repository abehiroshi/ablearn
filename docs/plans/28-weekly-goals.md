# 週間目標と今日の課題（自己選択型）

- 状態: 完了（2026-06-12）
- 触るファイル範囲: `src/lib/`（goals 新規）・`src/lib/storage.ts`（選択目標・週履歴）・`src/screens/HomeScreen.tsx`・目標選択UI
- 引き継ぎメモ（実装）:
  - ロジックは `src/lib/goals.ts`（カタログ16種・週進捗・日割り課題・週替わり適用・祝福）。すべて純関数でテストあり。
    選択状態は `AppState.goals`（active / next / weekStart / introDismissed。フィクスチャ v1-after-27 追加）
  - 進捗は既存記録のみから計算。「復習解消」は日単位履歴の制約上
    「正解ゼロで終えた日 → 後日の正解」を解消とみなす近似（コードのコメント参照）
  - 祝福は計画18に相乗り: ID `goal:<目標ID>:<週の月曜>` を celebrated に記録（週が替わるとIDが変わり再祝福可能）。
    達成判定は handleAnswer 内で「この解答を反映した状態」を組んで行う。バッジ棚は describeGoalMilestone で復元
  - 模擬テスト系目標（mock-1/2）の達成は finishMock で celebrated に記録するだけ（模試の結果画面にチップは出さない。
    ホームの目標カードで「たっせい！」表示）。「テスト日までにN回」は実装上「週内N回」に簡略化した
  - 週替わりの適用（next → active）はアプリ起動時の rolloverGoals。PWAを日曜夜から月曜まで開きっぱなしの場合、
    予約の適用が次回リロードまで遅れる（進捗バー自体は常に今週分を表示するので実害は小）
  - UI はホームに集約: 未選択＋未案内なら Abler の案内 → 選択UI（最大3・テスト登録中はテスト用が加わる）→
    目標カード（進捗バー＋今日の課題＋「来週から」予約表示）

## 目的

本人が選んだ週単位の目標から「今日の課題」を自動生成し、進捗率で達成感を作る。
押し付けの課題ではなく自己選択（自律性）を核にする。年間を通して使える選択肢を用意する。

## 構造（3層）

1. **目標カタログ**: プリセットから本人が選ぶ。**最大3個**（欲張ると全部が薄くなる）
2. **週の目標**: 週単位で判定。**週は月曜始まり**（学校のリズムと一致）
3. **今日の課題**: 週の目標から毎日自動生成。**残量÷残り日数で動的に再計算**する
   （週70問目標で月火さぼっても「今日は14問」と現実的に組み直す。1日の失敗を週の挫折にしない）

## 目標カタログ

**日常モード用**:
- 頻度系: 週に3日/5日学習する（streak の柔らかい版）
- 量系: 週に合計35/70/105問（1日平均5/10/15問相当）
- 復習系: 週に復習対象を10/20問解消する
- 幅系: 週に3教科以上に触れる
- 学習系: 週にレッスンを1/2本進める（予習）

**テストモード用**（テスト登録で提案が自動で切り替わる。常設の週目標の上に乗る）:
- 範囲達成度を50/80/100%まで上げる（18の達成度を使用）
- テスト日までに模擬テストを1/2回受ける
- 範囲内の復習対象をゼロにする

## ルール

- 進捗は04の履歴・dailyLog から計算（新規の記録は不要）
- 週目標の達成は18の節目システムで祝福する（同じ週の再祝福なし）
- **未達の週は静かに流す**（責めない・ペナルティなし。単調増加の原則と整合）
- 目標の変更はいつでも可、**適用は翌週から**（週の途中で下げて達成扱いにさせない）
- 初期状態は未選択。Abler が一度だけ「目標を選ぶと今日の課題が出るよ」と案内する

## 非スコープと破綻条件

- 数値の自由入力はやらない（プリセットのバリエーションのみ）→「1日1000問」のような壊れた契約を許さないため
- リマインダー通知はやらない（バックログの通知項目と一緒に判断）
- 親が目標を設定する機能はやらない → 本人が選ばない状態が続いたら検討

## 受け入れ条件

- 目標を選ぶとホームに「今日の課題」と週の進捗バーが出る
- さぼった翌日に課題が現実的な量に再計算されている
- 週目標の達成で祝福が一度だけ出る
- テスト登録でテスト用目標の提案に切り替わり、テスト終了で日常用に戻る
- 既存データ互換（フィクスチャ追加）・goals ロジックにテストがある
