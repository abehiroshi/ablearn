# 進捗バックアップ（エクスポート/インポート）

- 状態: 完了（2026-06-12）
- 触るファイル範囲: `src/lib/storage.ts`・`src/screens/StatsScreen.tsx`（＋必要なら `src/styles.css`）
- 引き継ぎメモ: `makeBackup`/`parseBackup` は `src/lib/storage.ts`。スキーマバージョン1。検証は外形のみ（app名・バージョン・xp/dailyLogの型）で、中身は計画どおり不透明に扱う。タブレット実機（iOS Safari/PWA）でエクスポートのダウンロード挙動を一度確認すること（download属性の扱いが環境依存）

## 目的

localStorage は端末変更・ブラウザのデータ削除で消える。streak や成績の消失はモチベに直撃するため、
利用が本格化する前に手動バックアップ手段を用意する。

## 機能

- Stats 画面の下部に「エクスポート」「インポート」を置く
- エクスポート: AppState 全体＋スキーマバージョンを JSON ファイルとしてダウンロード
- インポート: ファイル選択→内容のサマリ（XP・記録件数など）を見せて上書き確認→復元
- AppState の中身には立ち入らず、全体を不透明に入出力する

## 非スコープと破綻条件

- 自動同期・クラウド保存はしない → 複数端末で日常的に使い分けたくなった時点で破綻する。1人・1端末運用なら手動で足りる
- 部分インポート（マージ）はしない → 2端末の進捗を合流させたくなった時点で破綻する

## 受け入れ条件

- エクスポートした JSON を別ブラウザでインポートすると、XP・streak・成績がすべて再現される
- 不正なファイル（別アプリのJSON・壊れたJSON）を選んでもエラー表示のみで既存データは無傷
- インポートは確認を挟まずに上書きしない
