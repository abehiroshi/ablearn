# 実技4教科のコンテンツ追加

- 状態: 未着手
- 触るファイル範囲: `public/content/` のみ（アプリコードには触れない）
- 引き継ぎメモ: テスト範囲は「今は気にしない」方針（2026-06-11）。実装キューの最後尾。着手時に範囲・教科書を確認する

## 目的

期末テストは5教科＋実技4教科（音楽・美術・保健体育・技術家庭）で行われることが多い。
コンテンツ分離設計により、`index.json` への教科追加とセットJSONの配置だけで対応できる。

## 方針

- 対象: 音楽・美術・保健体育・技術家庭の4教科（中2・1学期期末の範囲から）
- 出題形式は既存の choice / input / flashcard で足りる（実技の定期テストは知識暗記が中心）。新形式は作らない
- 単元構成は教科書・学校の進度に依存するため、作成前に実際のテスト範囲を確認してから単元を切る
- まず1教科を従来どおり（セッションで生成して手で登録）作り、その作業の面倒さを見てから
  生成・検証パイプラインを作るか判断する（→ 判断したら別計画として起こす）

## 品質の注意（README の規約に従う）

- `id` はセット内で一意。既存 `id` の変更は履歴が切れるため禁止
- `input` の `answers` は漢字・ひらがな・別表記を網羅する
- 教科の `color` / `icon` は既存5教科と被らないものを選ぶ

## 非スコープと破綻条件

- 中間テスト用の実技コンテンツは作らない → 実技が中間にも出る学校だった場合は範囲を作り足す
- 生成パイプラインはこの計画に含めない → 4教科ぶん手作業でやって辛ければその時点で起こす

## 受け入れ条件

- `index.json` に4教科が追加され、Library・テスト登録の教科選択に表示される
- 各教科に最低1セットがあり、実際に解ける
- 既存5教科の表示・進捗に影響がない
