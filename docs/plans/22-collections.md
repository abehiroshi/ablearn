# コンテンツコレクション（URL分離）

- 状態: 未着手
- 触るファイル範囲: `src/`（App・lib/content・lib/storage）・`public/content/` の配置替え・`vite.config.ts`・`public/manifest` 関連・`.github/workflows/`（必要なら）
- 引き継ぎメモ: **未使用期間中に入れる前提の破壊的変更**（既存 localStorage キー・URLからの移行処理は作らない）。お子さんの利用開始前に完了させること

## 目的

同じアプリ（コード）で、中学教科書・英検・漢検のような独立したコンテンツの世界を
URLで分けて提供する。1コレクション = コンテンツ＋進捗が完全に独立した1つのアプリ体験。

## URL設計（全コレクションが対称にパスを持つ）

```
/ablearn/          → コレクション一覧のランディング（シンプルなリンク集）
/ablearn/chugaku/  → 中学教科書（現行コンテンツをここへ移動）
/ablearn/eiken/    → 英検（将来。コンテンツは別計画）
/ablearn/kanken/   → 漢検（将来。コンテンツは別計画）
```

- GitHub Pages は静的なので、ビルド時にコレクションごとの `index.html` を実パスに生成する
- コレクション一覧は `public/content/collections.json` で管理（id・名前・説明）

## 内容

- **コンテンツ**: `public/content/<collection>/index.json` に再配置。既存コンテンツは `chugaku/` へ移動
- **進捗の分離**: localStorage キーを `ablearn:<collection>:v1` に統一。
  既存 `ablearn:v1` からの移行はしない（未使用のため。残っていたテストデータは捨てる）
- **PWA**: コレクションごとにマニフェスト（名前・start_url）を分け、ホーム画面に別アイコンで
  インストールできるようにする（アイコンの色違い等は T系タスクで後から差し替え可能）
- **PWA名はコレクションごとに決める**。chugaku は **「中学生問題集」**（オーナー決定 2026-06-12）。
  英検・漢検の名前は各コンテンツ計画を起こすときに決める
- **バックアップ（02）**: エクスポートにコレクションIDを含め、別コレクションへのインポートを拒否する
- 機能側の分岐は「どの index.json を読むか」「どのキーに保存するか」の2点に閉じ込める

## 非スコープと破綻条件

- 英検・漢検の実コンテンツは含めない（コレクションの器だけ。中身は別計画で起こす）
- コレクション横断の共有要素（合算streak・共有スキン等）はやらない →
  「英検の日も streak を繋げたい」が出たら「共有プロフィール」として別設計を起こす
- 英検のリスニング（音声再生）は非対応のまま → 音声が必要になったら別計画

## 受け入れ条件

- `/ablearn/chugaku/` で現行の全機能・全コンテンツが動く
- ルート `/ablearn/` にコレクション一覧が表示され、chugaku に遷移できる
- コレクションごとに localStorage が分離されている（chugaku の解答が他に影響しない）
- PWA としてコレクション単位でインストールできる
- 既存の自動テスト（15）がすべて通る。互換フィクスチャはキー名変更に追従させる
- バックアップのコレクション不一致インポートが拒否される
