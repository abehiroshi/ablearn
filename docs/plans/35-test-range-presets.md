# テスト範囲のプリセット（学期テストの単元タグと一括チェック）

- 状態: 未着手
- 触るファイル範囲: `src/types.ts`（Unit.terms）・`src/lib/terms.ts`（新規・範囲導出）・
  `src/screens/TestSetupScreen.tsx`（ボタン行）・`scripts/validate-content.mjs`（terms 検証）・
  `public/content/chugaku/index.json`（既存単元へのタグ付け）
- 出典: オーナー発案。範囲選択の「全セットを1つずつチェック」する手間を、定期テスト名の
  ワンタップで解消する（推奨デフォルト＋本人の上書き優先の原則どおり）

## 目的

テスト登録の範囲選択に「1学期中間」「2学期期末」等のボタンを置き、押すとその範囲の単元の
セットが一気にチェックされる。学校ごとの実際の範囲ズレは、その後の個別チェックで調整する。

## 内容

- 単元タグ: `Unit.terms?: TermTest[]`。語彙は5つ
  （`1学期中間` / `1学期期末` / `2学期中間` / `2学期期末` / `学年末`）
- **配列にする理由**: 1学期をまるごと使う単元（例: 理科の化学変化）は中間・期末の両方の
  範囲に入る。設計時は1対1タグを想定したが、既存コンテンツのタグ付けで複数所属が
  実在したため配列にした（1要素が大半）
- TestSetupScreen の範囲カード先頭にボタン行。押すと:
  - その term を持つ全教科の単元の全セットを range に**追加**（union）
  - すでにその範囲が全部入りなら**外す**（トグル。個別チェックの状態は range が正のまま）
  - テスト名が初期値またはプリセット由来の名前のときだけ「◯学期◯◯テスト」に自動補完
- タグの無い単元（英単語・漢字など常設系、未タグの実技）はボタンに反応しない＝手動チェックのみ
- validator: terms の値が語彙内かを検証
- 既存単元へのタグ付けは標準進度（中間=10月中旬・期末=11月下旬。計画20の想定）で行う。
  実技4教科は1学期期末（中間テストに実技は無い）

## 非スコープと破綻条件

- 学校別・年度別のプリセット編集UIはやらない → タグは index.json の編集で直す
  （ズレたら個別チェックで吸収できるため、編集UIが必要になる頻度は低い想定）
- セット単位のタグはやらない → 範囲の粒度は単元で十分。単元内の一部だけ出る場合は個別チェックで外す
- 過去のテスト範囲の記憶・履歴はやらない

## 効果の見方

- テスト登録時のプリセット使用率と、プリセット後の個別調整の量（多ければタグの精度を直すシグナル）

## 受け入れ条件

- ボタンで該当単元のセットが一括チェックされ、個別に外せる（既存の toggleSet が効く）
- もう一度押すと範囲が外れる（トグル）
- タグの無い単元がボタンに反応しない
- terms の語彙外の値を validate:content が検出する
- 既存の範囲選択・保存の動作に影響がない
