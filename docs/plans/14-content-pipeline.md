# コンテンツ生成・検証パイプライン

- 状態: 完了（2026-06-12）
- 触るファイル範囲: `scripts/`・`docs/`（生成手順書）・`public/content/`（生成物）
- 引き継ぎメモ: 生成手順書は `docs/content-generation.md`（プロンプトテンプレート込み・自己完結）。検証は `npm run validate:content`（＋ `--links` で死活。403/405/429 は bot 対策の可能性があるため注意扱い）。実例は理科「化学反応式の係数」概念ファミリー（レッスン＋3変種）。初仕事だった実技4教科（03）は手作業で先行完了済みのため、次のコンテンツ作業（20の2学期分・S系）からこの手順書を使う

## 目的

概念ラダー前提のコンテンツ（1概念 = レッスン＋変種数問＋ヒント段階＋難易度＋links）は
手作業では量が追いつかない。生成と検証を仕組み化し、コンテンツの質と量を両立する。

## 内容

- **生成**: 入力は「教科・単元・範囲（教科書の該当箇所）」。出力は概念ごとの問題ファミリー一式
  （レッスン・変種問題・段階ヒント・難易度タグ・links 候補）のセットJSON。
  生成はセッション（AI）で行い、手順をプロンプトテンプレートとして `docs/` に置く
- **検証スクリプト**（`scripts/`）: スキーマ準拠・セット内id一意・order問題の並びの一意性・
  input の answers 網羅の形式チェック・links の死活チェック
- **レビュー**: 生成物は登録前に内容の正確性を人が（または Breaker レビューで）確認する。
  links は親が選定したもの以外を含めない
- **初仕事**: 実技4教科（[03](03-jitsugi-content.md)）のコンテンツ作成をこのパイプラインで行う

## 非スコープと破綻条件

- アプリ内での生成はしない（静的PWAの外）
- 検証は形式チェックまで。内容の正誤までは自動判定しない → 誤答コンテンツが混入したらレビュー工程を強化する

## 受け入れ条件

- 1単元を入力に、検証を通る概念ファミリー一式が生成できる
- 検証スクリプトが壊れたJSON・id重複・死リンクを検出して落とす
- 生成手順書を見れば、このセッションの文脈なしで誰でも（どのセッションでも）同じ品質で生成できる
