# マスコットスキンの解放（ご褒美の最上位）

- 状態: 未着手
- 触るファイル範囲: `scripts/abler-build.mjs`（複数シート対応）・`src/components/Abler.tsx`・`src/lib/storage.ts`（解放・選択状態）・着せ替えUI（Stats か Home）
- 引き継ぎメモ: 最初のスキンの設定画シートは外部制作（`assets/TASKS.md` の T-001）。仕組み＝受け入れ口を先に作っておく

## 目的

大きな節目だけで解放される希少なご褒美として、Abler のスキン（着せ替え）を用意する。
アプリ全体の見た目が変わるため、バッジより一段上の報酬になる。

## 内容

- スキン = 設定画シート1枚。既存のアセットパイプライン（abler-build.mjs）を複数シート対応にし、
  シートを追加すれば同じポーズ構成で切り出される形にする
- 解放条件は [18](18-metrics-and-milestones.md) の節目システムを使う（例: 教科達成度100%・streak 30日）。
  ショップ・XP消費にはしない（達成→解放の直接対応）
- 着せ替えUI: 解放済みスキンから選択。**未解放はシルエット＋解放条件を表示**（次の目標の可視化を兼ねる）
- 解放・選択状態は進捗データに保存し、バックアップ（02）に含める

## 非スコープと破綻条件

- スキンごとのポーズ数はメインと同構成（一部ポーズだけのスキンは作らない）→ シートが揃わないスキンを入れたくなったらフォールバック（メインのポーズで代用）を検討
- スキンの販売・期間限定はやらない（1人用）

## 受け入れ条件

- 解放条件を満たすと祝福とともにスキンが解放され、着せ替えできる
- 未解放スキンの解放条件が見える
- スキン未設定（メイン）の既存表示が一切変わらない
- 解放・選択状態がバックアップに含まれる
