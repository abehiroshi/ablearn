# 概念ラダーのスキーマ拡張

- 状態: 完了（2026-06-12）
- 触るファイル範囲: `src/types.ts`・`src/lib/content.ts`・`README.md`(コンテンツ規約の追記)
- 引き継ぎメモ: スキーマ＋変換（`choiceAsInput` in `src/lib/quiz.ts`）＋表示（単元リンク=Library、問題リンク=解答後のフィードバック内）を実装済み。実例: 理科「化学変化と原子・分子」単元にぽにょんのチャンネルURL（単元別の動画URLへの差し替えは親が選定後に行う）、数学 式の計算 q1/q3 が concept "shiki-doruiko" のラダー（q1 は answers 併記で両形式可）。出し分けフラグは `QuizItem.asInput`（QuizScreen 対応済み。現状どこも立てない。立てるのは12）

## 目的

問題を単品ではなく「同じ概念を確認する問題の階段（概念ラダー）」として扱える土台を作る。
以後のコンテンツはすべてこの形で作成できるようになるため、適応出題（[12](12-mastery-engine.md)）より先に入れる。

## スキーマ（すべて任意フィールド・既存コンテンツ互換）

- `concept?: string` — 問題に概念IDを付与。同じ concept の問題群が1つのラダー
  （例: 易しいchoice → input → 応用、の変種が同じ concept に並ぶ）
- choice 問題に `answers?: string[]` を追加 — 受理表記があれば、同じ問題を
  習熟度に応じて **choice（4択）/ input（自力入力）どちらの形式でも出せる**
- `links?: {label, url}[]` — 単元（index.json の Unit）を主な置き場とし、問題にも任意で持てる。
  授業で使われている動画（例: 理科のぽにょん）や解説サイトへの導線

## 運用ルール（README に追記する）

- 問題文のバリエーションは実行時生成ではなく、作成時に変種を複数作って同じ concept に並べる
- links は親が選定したものだけを入れる

## 非スコープと破綻条件

- 実行時の数値テンプレート生成（パラメータ化問題）はしない → 変種の事前作成で量が追いつかなくなったら再検討
- リンクの埋め込み再生はしない（外部タブで開くだけ）

## 受け入れ条件

- 新フィールドを使ったセットが読み込めて動く
- 既存コンテンツ（フィールド未設定）の表示・進捗が一切変わらない
- choice+answers の問題が両形式で出題・正誤判定できる（出し分けロジック自体は12）
