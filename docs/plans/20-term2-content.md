# 中2・2学期コンテンツの追加

- 状態: 未着手
- 触るファイル範囲: `public/content/` のみ＋生成手順書（`docs/`）。アプリコードには触れない
- 引き継ぎメモ: アプリ機能のキューとファイル競合しないため**いつでも並行で着手可**

## 目的

中2・2学期の単元（5教科）を追加する。11完了後の新フォーマットを最初から標準装備し、
作成の過程で生成手順をテンプレート化する（[14](14-content-pipeline.md) の生成手順書を兼ねる）。

## 準拠する教科書（板橋区立中学校・令和7〜10年度採択）

- 数学・理科・英語（NEW HORIZON）・技術家庭: 東京書籍
- 国語・書写: 三省堂 ／ 地理・歴史・音楽: 教育出版 ／ 公民・美術・道徳: 日本文教出版
- 地図: 帝国書院 ／ 保健体育: 大修館書店
- 単元の区切り・順序・用語はこの採択教科書の標準的な進度に沿って作成する
  （各教科書の目次・単元構成は作成セッションが調べて決める。学校固有の進度ズレは使いながら調整）

## 方針

- **全教科標準装備**: 難易度タグ＋段階ヒント（07の品質を新規分は最初から）
- **数学・理科を優先**: concept タグ（choice/input両用の変種を含む概念ラダー）
- **レッスンは数学の2学期最初の単元（一次関数）から1本**。手応えを見て拡大
- 2学期分を先に作り切る。3学期分は手順書が確立してから（このファイルに追記せず計画21として起こす）
- 最初の1〜2教科で生成手順をテンプレート化して `docs/` に残し、残りの教科はそれをなぞる
- 検証は `npm run validate:content`（15で整備済み）を必ず通す

## 非スコープと破綻条件

- 実技4教科の2学期分は含めない → 2学期の期末が近づいたら03の方式で追加
- 学校の実際の進度・テスト範囲への厳密な一致は求めない → テスト登録（01）の範囲選択で吸収する

## 受け入れ条件

- 5教科それぞれの2学期主要単元が `index.json` に登録され、アプリで解ける
- 新規セットはすべて難易度・ヒント付き。数学・理科は concept ラダーを含む
- `npm run validate:content` が通る
- 既存コンテンツ・進捗に影響がない
- 生成手順書が `docs/` にあり、3学期分（計画21）が手順書だけで作れる状態になっている
