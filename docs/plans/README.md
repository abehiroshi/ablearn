## 後で計画化（着手が見えたら起こす）

- ゴースト並走（**保留**・オーナー判断 2026-06-12）— 再演習時に「前回の自分の正答率」を並走表示する
  成長実感装置（ゲーム調査 A-2 が最有力と評価）。保留理由: 適応出題・変種により周回間の問題セットが
  揃わず正答率を比較できない。起こすときは模擬テスト再戦・再戦セット（計画30）など固定セットへの
  限定から設計し、同段変種ローテーション（下記）との整合を先に決める。
  正答率のみ（解答時間は見せない・12の方針と同じ）・「負け」を見せず並走/接近の演出にする
- 同段変種のローテーション出題 — 適応出題（mastery.ts の pickVariant）は同じ段の先頭変種しか
  選ばないため、計画23で導入した自己説明・誤答修正・faded 変種は通常演習に出ない
  （模擬テスト・レッスンでは出る）。新基準コンテンツが増えてきたら、同段の変種を
  正答履歴でローテーションする小改修を計画化する
- 個人版予想問題 — S系タスク（学校テスト写真）が数回分貯まったら、出題傾向から予想セットを生成（調査レポート B-5）
- 「あとで見直す」ブックマーク
- 学習リマインダー通知（保留: 端末のPWA通知制約の確認後）
- 親ビュー（保留: Stats＋可視化で足りるか様子見)
- AI動的ヒント・手書き認識（静的PWAの外。必要になったら）

    （URLパラメータでの事前入力 → Web Share API → クリップボードの3段構え。APIキー・サーバー不要で現設計を崩さない）
