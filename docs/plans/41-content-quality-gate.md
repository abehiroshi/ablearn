## 目的

「高水準」を感覚でなく数値で定義し、CI で守られる状態にする。
そのうえで計画23以前に作られた旧セット（主に1学期分）を新基準へ改修する。

## 内容

### 1. 品質基準の数値化（validate:content に閾値チェックを追加）

chugaku コレクションの演習問題（choice / input / order）を対象に:

- explanation はステップ＋理由の形式で 100%
- 2段ヒントを 100%
- concept 付与率: 教科ごとに 85% 以上（flashcard は対象外）
- 難易度3（応用）: 教科ごとに演習問題の 15% 以上
- 習熟概念ごと: 変種5以上・input 可能（input 型 or answers 付き choice）2以上・
  難易度 stage0/1/2 の展開あり ※この概念別チェックは計画42〜45の完了までは警告扱いとし、
  45完了時にエラーへ昇格する（移行期に CI を赤くしない）
- 閾値はコレクション別に持つ（kanken は flashcard 中心で性質が違うため適用しない）

### 2. 旧セットの改修

- 1学期分を中心に、上記基準を満たすよう explanation・ヒント・answers・難易度タグを補強
- 問題 id は変えない（履歴が切れるため）。既存問題の改修は文言・メタデータの追補に留め、
  作り直しが必要な場合は新 id の変種として追加する

## 非スコープと破綻条件

- 問題の増量は計画42〜45で行う（本計画は基準とゲートと既存分の改修）
- 破綻条件: 既存問題の id 変更・削除は解答履歴と習熟度を切る。改修は追補のみとする

## 受け入れ条件

- `npm run validate:content` が新基準のチェックを実行し、基準値がスクリプトに数値で残っている
- chugaku の全既存セットが基準（概念別チェック以外）を満たして CI が通る
- docs/content-generation.md に基準が明文化され、以後のセットは作成時点で基準を満たせる
