## 目的

単元の習得に必要な解説と演習の全体量と現在地を、数値ではなく「一本道のマス目を進む」絵で見せる。
「あと何マスでこの単元が終わる」が一目で分かる状態を作る。

## 内容

- **1マス = 1セット**（レッスンも演習も）。マス列は `index.json` の単元内セット配列の順そのまま
  （新しいデータ構造は持たない。計画14/20でセットが増えるとトラックが自動で伸びる）
- マスは2状態:
  - **通過** = そのセットを1回完走（setRecords）
  - **クリア** = 達成条件を満たした（ベストスコア80%以上目安。18の達成度の流用）
- **踏破は単調増加**: 一度進んだマスは戻らない・暗転しない（18の原則と同じ。SRSで習熟の鮮度が
  落ちても、それは復習リスト側の仕事でありマスには反映しない）
- **ロックしない**: 道順は推奨であってロックではない。次のマスをデフォルトフォーカスで光らせるが、
  先のマスも踏破済みのマスもタップで挑める（AGENTS.md プロダクト設計原則）
- **先のマスは一部未開示**: 次のマスだけ名前を出し、その先は「？」やシルエットで存在だけ見せる
  （総量は見えるが中身は見えない = U字曲線の応用）
- 置き場所: Library の単元ビューを本体（縦の一本道）、ホームの進行中単元カードにミニ版（●●○??）
- 現在地に Abler を立たせる（スキン19のポーズ・スキンがそのまま乗る）
- 単元の全マスクリアは18の祝福体系に相乗り

## 非スコープと破綻条件

- 分岐・寄り道マスはやらない（一本道だから現在地が一目で分かる。分岐が欲しい単元構造が出たら再設計）
- マスの中の進捗（レッスン内の何ステップ目か等）はすごろくに出さない → レッスン画面内の既存表示に任せる
- 1セットしかない単元は1マスの道になるが許容（コンテンツが増えれば自然に解消。20より前に入れると
  「道が伸びた」こと自体を新コンテンツ投入の演出にできる）

## 効果の見方

- 単元ビューの閲覧頻度と、そこからのセット着手率（「あと2マス」が着手のきっかけになっているか）

## 受け入れ条件

- index.json から単元ごとのトラックが導出され、通過/クリア/未到達が正しく表示される
- どのマスもタップで挑める（ロックがない）
- 踏破状態が単調増加であること（後からスコアが下がってもマスが戻らない）がテストで確認できる
- セットの追加でトラックが自動で伸びる（index.json の変更だけで反映される）
