# 国語の概念体系化と増強

- 状態: 未着手
- 触るファイル範囲: `public/content/chugaku/japanese/`・`public/content/chugaku/concepts.json`・`public/content/chugaku/index.json`。アプリコードには触れない
- 引き継ぎメモ: 計画41の完了後・42/43と同型。
  ベースライン: 61問・concept率 20%・ヒント率 67%・answers率 66%・難易度3 11問・漢字1セット

## 目的

国語は文法・古典という「手順のある領域」があるのに concept 率20%で、
漢字は1セットしかない。文法を「育つ領域」に、漢字を「量で回す領域」にする。

## 内容

- **文法の概念ファミリー化**: 用言の活用（動詞・形容詞・形容動詞）・助詞/助動詞の識別
  （「ない」の品詞識別・「れる/られる」の意味の区別 等）を concepts.json に登録し、
  前提を宣言する（例: 助動詞の識別 ← 用言の活用）。手続き概念としてフル変種
  （自己説明・誤答修正・faded）を備える
- **古典の概念化**: 歴史的仮名遣いの読み替え・係り結び・現代語訳の要点（平家物語・徒然草・漢詩）
- **漢字の増強**: 中2配当を学期別セットに分けて計200字目安まで増やす。
  読み（input）と書き（flashcard＋手書き余白の自己採点方式＝kanken 計画38と同方式）の両方向
- **全演習問題に concept を付与**し、計画41基準の変種構成へ拡張

## 非スコープと破綻条件

- 読解問題（文章題）は扱わない（長文の出題形式が現アプリにない。必要になったら別計画）
- 破綻条件: 漢字の書きを input にすると IME 経由で「変換して選ぶだけ」になり書く練習にならない。
  書きは flashcard＋自己採点方式に倒す（kanken で実証済みの方式を踏襲）

## 受け入れ条件

- 文法・古典の演習問題の concept 付与率 85% 以上・計画41の教科別基準をすべて満たす
- 漢字が学期別セットで計200字規模・読み書き両方向ある
- 文法概念で写経段→choice→input の昇格が実際に動く（手元確認）
