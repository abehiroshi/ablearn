# ヒント開示方法の選択（すこしずつ / ぜんぶ見る）

- 状態: 未着手
- 触るファイル範囲: `src/screens/QuizScreen.tsx`（ヒントUI）
- 出典: ゲーム調査 A-3（U字型好奇心曲線: 段階開示が探究を生む）と教育調査 C-1（worked-example:
  初学者には全部見せる方が良い場合がある）が衝突するため、**どちらが良いかを決め打ちせず本人に選ばせる**（オーナー決定）

## 目的

ヒントの開き方を本人に選ばせ、「あと一歩で分かりそう」の探究と「全部見て理解したい」の
worked-example 的学習を両立する。選択自体が自律性を満たす。

## 内容

- ヒントボタンを押した時に**「すこしずつ」/「ぜんぶ見る」の2択を毎回提示**する
- すこしずつ = 既存の段階開示（07のまま、弱→強を1段ずつ）
- ぜんぶ見る = hints 全段を一括開示
- 習熟度上の扱い: ぜんぶ見る＝「ヒントを最後まで使った」（12の降格ルールどおり）。
  すこしずつでも最終段まで開けば同じ扱い（開示方法ではなく到達段数で判定 — 既存ルールを変えない）
- XPは07の体系のまま（ヒント使用後の正解 +5）
- 解答履歴に開示方法（段階/フル）を記録する

## 非スコープと破綻条件

- 選択の記憶（前回選んだ方をデフォルトにする等）はやらない → 毎回の能動的な選択に意味がある。
  毎回の2択が煩わしいという声が出たら記憶式を検討
- ヒントのない問題では従来どおりボタン自体を出さない

## 効果の見方

- 段階/フルの選択比率（習熟が進むと段階選択が増えるかは観察ポイント）
- 開示方法別の後日定着（同 concept の後日正答率）

## 受け入れ条件

- ヒントボタンで2択が出て、それぞれ正しく開示される
- フル開示後の正解が習熟度・XPに既存ルールどおり反映される
- 履歴に開示方法が記録される
- ヒント未設定の問題の動作が変わらない
