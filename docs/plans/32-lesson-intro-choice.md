# レッスン冒頭の入り方選択（問題をチラ見 / 解説から）

- 状態: 未着手
- 触るファイル範囲: `src/screens/LessonScreen.tsx`
- 出典: ゲーム調査 A-1（PvZ: 説明より先にまず1回やらせる）と教育調査 C-1（worked-example:
  初学者は解説が先）が衝突するため、**決め打ちせず冒頭で本人に選ばせる**（オーナー決定）

## 目的

レッスンの入り口に「まず問題を見てみる」選択肢を作り、「何を解けるようになるための解説か」を
先に体感してから読めるようにする。テストモードとレッスンモードの明確な選択は現状のまま維持する。

## 内容

- レッスン開始時に**「まず問題をチラ見」/「解説から読む」の2択を提示**する
- チラ見 = そのレッスン内の最初の問題（代表1問）を**採点なしで**眺める。
  解答してみてもよいが、正誤は記録せず XP もなし（「解けなくて当然」の地点なので痕跡を残さない）
- チラ見後は通常のステップ列（解説カード→易問…）に合流する
- 解説から読む = 従来フローそのまま（無変更）

## 非スコープと破綻条件

- チラ見問題の専用作問はやらない（既存ステップ列の問題を流用。流用できる問題がないレッスンでは選択肢自体を出さない）
- レッスン途中での「先の問題を覗く」機能はやらない → 欲しくなったら別計画
- 8語ルール等の解説文の簡潔化はこの計画に含めない → コンテンツ生成手順書（23）側の話。
  計画20（2学期コンテンツ）の生成時に文言基準として反映するか個別に判断

## 効果の見方

- チラ見の選択率とレッスン完走率（チラ見選択時に完走率が上がるか）

## 受け入れ条件

- レッスン開始時に2択が出て、チラ見が questionStats・XP に一切影響しない
- 「解説から読む」を選んだ場合の挙動が従来と完全に同じ
- チラ見できる問題がないレッスンでは従来どおり始まる
