# つまずき誘導と予習フロー

- 状態: 未着手
- 触るファイル範囲: `src/lib/recommend.ts`・`src/screens/`（QuizScreen・HomeScreen）
- 引き継ぎメモ: なし

## 目的

わからない問題を繰り返し外し続ける体験をなくす。新しい単元では演習の前に
レッスン（[10](10-lesson-mode.md)）へ誘導し、予習にも使えるようにする。
新機能というより既存部品（レッスン・ヒント・おすすめエンジン）の接続。

## 内容

- **つまずき検知**: 同一概念で不正解が続く／ヒントを使い切っても不正解のとき、
  繰り返し出題を止めて Abler が「この単元のレッスンをやってみる？」と誘導する（該当レッスンがある場合）
- **予習フロー**: 進行中単元に演習履歴がほぼない場合、おすすめはドリルではなくレッスンを先に出す。
  順序は「レッスン → 易しい問題 → 演習 → 応用」（習熟度エンジン [12](12-mastery-engine.md) の段がそのまま使える）
- レッスンがない単元では、単元の links（[11](11-concept-schema.md)）があればそれを提示する

## 非スコープと破綻条件

- レッスンの自動生成はしない（[14](14-content-pipeline.md) の領分）
- つまずき検知の感度調整UIは作らない → 誘導がうるさい/鈍いと感じたら閾値を直接調整

## 受け入れ条件

- 新しい単元を選んだ直後のおすすめ先頭がレッスンになる（レッスンがある場合）
- 同じ問題を外し続けたとき、3回目以降は誘導が出る
- 誘導を無視して演習を続ける選択も常にできる
