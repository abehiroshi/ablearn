# 漢検 準2級 合格逆算の増強（読みカバレッジ中心）

- 状態: 未着手
- 触るファイル範囲: `public/content/kanken/`（セット追加・index.json）。スクリプト・アプリコードは触らない
- 引き継ぎメモ: なし

## 目的

計画39の増強（22セット228問・カバー115/328）を「最短の学習時間で合格」へ向けて仕上げる。
合格140/200点を分野別に逆算し、点数効率（配点÷習得コスト）の高い分野へ集中投資する。

## 点数設計（このプランの根拠）

| 分野 | 配点 | 目標 | 根拠 |
|---|---|---|---|
| 読み | 30 | 9割 | 認識のみで習得コスト最低。配当字中心の出題 |
| 四字熟語（書き＋意味） | 30 | 8割 | 有限リスト。書きと意味が同じ語彙から出る二重取り |
| 書き取り | 50 | 7割 | 範囲が広く裾が長い。定番語で妥協する |
| 対義語類義語・同音同訓 | 各20 | 8割 | 定番ペア・パターンが有限 |
| 部首・誤字訂正・送りがな | 各10 | 現状維持 | 配点小。質の修繕のみ・量は増やさない |

→ 合計約155点。書き取り満点や328字完全制覇は学習時間の無駄として捨てる。

## 内容（優先順）

1. **読みカバレッジ・シリーズ（最大の増強）**: 未出題213字を読みセットで体系的に消化する。
   「読み 配当字1〜n」として1問に未出題字1〜2字を含む熟語で作問（12問×6〜8セット）。
   `npm run coverage:kanken` を見ながら熟語頻度の高い字から埋め、カバー率70%超を目安にする。
   「璽・朕・虞・詔」級の“読み問題の定番だが熟語が少ない字”も読みなら安価に出せる
2. **質の修繕「基本3」**: 部首・熟語の構成・誤字訂正に本番水準の基本3を1セットずつ（計画39の残課題①）。
   量は増やさない。特に誤字訂正は配点10点に対し作問コスト最大（文脈一意性のBreaker指摘が集中）なので量産しない
3. **四字熟語 書き3・意味3**: 頻出四字熟語の拡充（+20問、計36→56問規模）
4. **書き取り 基本4〜5**: 「書き誤りやすい定番語」中心に+25問前後（計65問規模）。
   準2級の書き取りは下位級配当字の定番語も多いため、配当字カバレッジに固執しない
5. **対義語類義語3・同音同訓3**: 定番ペアの拡充（各+10問）

完了時の総量目安は約380問。1周2〜3時間×3周＝8時間前後で回せる規模に収め、
コンテンツ過剰で学習時間が延びる事態を避ける（これ以上の量的増強はしない）。

## 生成・検証の規約

- 生成は `docs/content-generation.md` の「漢検コレクション（kanken）の注意」に従う
  （形式の使い分け・誤字訂正の同音一字・四字熟語の実在確認・転載禁止）
- セット追加ごとに `npm run validate:content` ＋ 漢字知識のBreakerレビュー
  （計画38・39の実績観点: 部首の覚え方・誤字訂正の文脈一意性・配当レベル・表外読みの排除）

## 非スコープと破綻条件

- 模擬テストはやらない（受検日が決まったら別計画。計画39の方針を継承）
- 328字の完全網羅は完了条件にしない（読み主体で85%程度まで上がれば道具として十分）
- 部首・誤字訂正・送りがなの量的増強はしない（配点が小さい）
- 頻出度データそのものはリポジトリに取り込まない（著作権。語の選定の参考にするだけ）
- 他の級（4級・3級・2級）は対象外
- 破綻条件: 読みセットを未出題字消化だけで埋めると低頻出の不自然な熟語が混ざり質が落ちる。
  1セット内の「未出題字消化問題」は8割までとし、残りは頻出定番語で埋める

## 受け入れ条件

- `npm run coverage:kanken` のカバー率が70%（230/328）以上
- 部首・熟語の構成・誤字訂正に本番水準の「基本3」があり、下位級寄りwarningの指摘が再発しない
- 四字熟語が書き3・意味3（計56問規模）、書き取りが計65問規模、対義語類義語・同音同訓が各3セットある
- 追加分すべてに validate:content ＋ 漢字知識のBreakerレビューを実施し、major 0 で完了
- chugaku コレクションに影響がない（kanken のコンテンツファイルのみの変更）
