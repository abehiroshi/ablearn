# 「わからない」選択肢

- 状態: 完了（2026-06-12）
- 触るファイル範囲: `src/screens/QuizScreen.tsx`・`src/components/QuestionViews.tsx`・`src/lib/storage.ts`（履歴への記録）
- 引き継ぎメモ: ボタンは `onGiveUp` プロップを渡した画面だけに出る設計（通常クイズのみ。模擬テストとレッスンには渡していない。レッスンは間違えてもその場で解説が出るため不要と判断）。履歴は `QuestionDayStat.dontKnow` に不正解と区別して記録（12・13はこれを読む）。再出題は不正解と同じくセッション内キューの末尾へ。受け入れ条件は storage.test.ts に自動テスト化済み

## 目的

4択は25%が当て推量で正解してしまい、習熟度（[12](12-mastery-engine.md)）の推定を汚染する。
「わからない」を正直に選ぶ方が当て推量より得になるよう動機を揃え、データの精度を上げる。
12より先に入れる（習熟度エンジンが最初からきれいなデータで動くため）。

## 内容

- 全形式（choice / input / order）に「わからない」アクションを置く
- 選んだときの体験を「罰ではなく学びへの近道」にする:
  - 即座に解説とヒントを表示する
  - Abler は責めない反応（kangaechu / ganbare 等）
  - 取組数にはカウントする（やった事実は減らさない）。XPは0だが減点もなし
- 解答履歴（04）に「わからない」を不正解と**区別して**記録する
  （12の習熟度推定・[13](13-struggle-and-preview.md)のつまずき検知が利用する）
- 習熟度上は正直な未習熟シグナルとして扱う（当て推量の誤答と同等の降格。正解扱いにはしない）

## 非スコープと破綻条件

- 模擬テストには出さない（本番のテストに「わからないボタン」はないため。白紙=不正解で扱う）
- 「わからない」の連打で解説だけ集める使い方は防がない → 乱用が見えたら出題間隔で抑制を検討

## 受け入れ条件

- 「わからない」を選ぶと解説・ヒントが表示され、次の問題に進める
- 履歴に不正解と区別されて記録される
- 模擬テスト中には表示されない
- 既存の正誤フロー（XP・streak・復習リスト）が壊れない
