# 習熟度エンジン（昇降格・適応出題・SRS）

- 状態: 未着手

- 引き継ぎメモ: なし

## 目的

正誤・ヒント使用・経過日数の実績から概念ごとの習熟度を推定し、
「正解し続けたらより難しく、つまずいたら易しく」を自動で行う。
難易度タグ（自己申告）は出発点の配置に使い、実際の昇降格は実績が決める。

## 昇降格ルール（v1）

- **昇格**: その概念で「ヒントなし正解」が2回連続、かつ前回正解から日数が空いていること
  （直後の連続正解は短期記憶。間隔をあけた正解を定着の証拠とする = SRS の考え方をここに統合）
- **降格**: 不正解、またはヒントを最後まで使った正解
- 段の構成: choice形式 → input形式 → 高難易度の変種（[11](11-concept-schema.md) の concept ラダー上を移動）
- 解答時間は評価に使わない（速さを評価すると雑に答える誘因になる）。表示・分析のみ

## 出題への反映

- おすすめエンジンの優先度に「定着確認の時期が来た概念」（間隔反復）を加える
- 出題時、概念の現在段に応じた形式・難易度の問題を選ぶ
- concept 未設定の既存問題は従来どおりの出題（後方互換）

## 非スコープと破綻条件

- IRT等の統計モデルは使わない → 1人用・この問題数では単純な階段で足りる。複数ユーザー化や数千問規模で再検討
- 習熟度の手動調整UIは作らない → 推定が体感とズレる声が出たら検討

## 受け入れ条件

- ヒントなしで間隔をあけて正解を重ねると、同じ概念がより難しい形式で出るようになる
- 不正解が続くと易しい段に戻る
- 既存データ（04の履歴・questionStats）から初期習熟度が計算でき、localStorage 互換が保たれる
