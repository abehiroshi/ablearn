# 難易度と段階的ヒント

- 状態: 完了（2026-06-12）
- 触るファイル範囲: `src/types.ts`・`src/components/QuestionViews.tsx`・`src/screens/QuizScreen.tsx`・`public/content/math/`（遡及タグ付け）
- 引き継ぎメモ: ヒントUIは QuizScreen 内（模擬テストには出ない）。数学4セット34問すべてに difficulty/hints 付与済み。模擬テストの難易度バランス出題（応用3割）も有効化済み。他教科のタグ付けはコンテンツ作業（03など）の流れで順次

## 目的

「わからないとき」の救済（段階的ヒント）と、出題・分析の軸になる難易度メタデータを導入する。

## スキーマ（どちらも任意フィールド・既存コンテンツ互換）

- `difficulty?: 1 | 2 | 3`（基本/標準/応用）。未設定は標準（2）扱い
- `hints?: string[]`。弱→強の順（例: 1段階目「考えるポイント」→ 2段階目「計算過程の途中まで」）

## UI

- 「ヒント」ボタンで1段階ずつ開示。Abler の `kangaechu` / `hirameita` ポーズで見せる
- ヒント使用後の正解は +5 XP（リトライ正解と同額。1発正解 +10 の体系に新区分を増やさない）
- ヒント使用は解答履歴（[04](04-answer-history.md)）に記録する（予約済みフィールドの有効化）
- 模擬テスト（[06](06-mock-test.md)）の難易度バランス出題をここで有効化

## 遡及タグ付け

- 数学の全セットに難易度・ヒントを先行で付与する（手書き余白・模擬テストとの相乗効果が最大の教科）
- 他教科は今後のコンテンツ作業（実技追加など）の流れで順次

## 非スコープと破綻条件

- ヒントは静的テキストに限定。本人の答案を見た動的な助言は静的PWAの外 → 欲しくなったら別計画
- ヒントのない問題ではボタンを出さない（全問題への網羅は求めない）

## 受け入れ条件

- ヒントが段階どおりに開き、使用後の正解XPが +5 になる
- 難易度・ヒント未設定の既存問題が今までどおり動く
- 数学の全セットにタグが付き、JSONとして正しく読み込める
